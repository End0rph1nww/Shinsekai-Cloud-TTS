# Cloud TTS 插件更新日志

## 0.11.0 (2026-05-24)

- **GPT SoVITS Cloud provider**: 新增 `gpt-sovits-api`，面向自部署 GPT-SoVITS `api_v2.py` 服务，支持 `/tts`、`/set_gpt_weights` 和 `/set_sovits_weights`。
- **legacy API 自动兼容**: 如果服务器启动的是 GPT-SoVITS `api.py`，插件会从 `api_v2.py` 路由自动退回 `POST /set_model` 和 `POST /`，适配仅提供 `/set_model` 的 V2Pro2025 等部署。
- **多版本模型路径**: 不限制权重文件后缀或 GPT-SoVITS 版本；V1/V2/V2Pro/V2Pro2025/V2ProPlus/V3/V4/自定义模型均按服务器路径传给 API。
- **角色级 GSV 配置**: 插件设置页可为每个角色单独保存服务器参考音频路径、GPT 权重路径、SoVITS 权重路径、参考文本、参考语言和文本语言。
- **绕过本地表单校验**: GPT SoVITS Cloud 路径保存在插件私有配置中，不复用主程序角色表单的本地文件/目录字段，避免云端 GSV 部署卡在地址校验。
- **V4 参数入口**: 设置页新增 GPT-SoVITS 分句、输出格式、`sample_steps` 和 `super_sampling` 配置。

## 0.10.1 (2026-05-15)

- **模型精简**: MiniMax 模型列表移除旧代 speech-02 / speech-01 系列，仅保留 speech-2.8 / speech-2.6 两代；Qwen 模型收束为单一 VC 模型 `qwen3-tts-vc-2026-01-22`，合成与声音复刻统一使用该模型。
- **编辑视图独立**: 设置页新增「TTS Provider 配置」下拉框，切换编辑 MiniMax / Qwen3 配置不再影响主 API 页当前选中项；model 和 default_voice_id 按 provider 分别写入 api.yaml。
- **Voice 数据分离**: voice_id_map、voice_id_versions 和 default_voice_id 按 provider 分目录存储（`minimax-tts/voices/` 和 `qwen-tts/voices/`），旧版共享 voice 数据自动迁移到 MiniMax 目录。
- **声音复刻端点修复**: Qwen 声音复刻跟随用户配置的 Base URL，不再硬编码国内端点，避免国际端点复刻的 voice 在合成时 400 报错。
- **声音复刻参数修复**: `preferred_name` 改为纯 ASCII 字母格式（映射 hex hash 到字母表），`target_model` 固定使用 VC 模型 `qwen3-tts-vc-2026-01-22`，修复 DashScope API 400 报错。
- **系统音色移除**: Qwen 设置页移除系统音色下拉框（Cherry、Serena 等 47 种内置音色），voice 统一通过声音复刻生成自定义音色。
- **文案更新**: 使用说明和错误提示移除系统音色相关描述。

## 0.10.0 (2026-05-15)

- **Qwen3 TTS 接入**: 新增 Qwen3 TTS（DashScope / 百炼）作为第二个 TTS provider，注册为 `qwen-tts`，支持语音合成和声音复刻。
- **QwenTTSAdapter**: 新增适配器类，通过 DashScope HTTP API 实现 `generate_speech()` 和 `create_cloned_voice_from_file()`。
- **Qwen 系统音色**: 配置页新增系统音色下拉框，支持 Cherry、Serena、Ethan、Chelsie 等 40+ 种内置音色，无需上传参考音频即可使用。
- **Qwen 合成语言**: 新增合成语言（language_type）下拉框，支持中文、英语、日语、韩语、法语等 10 种语言。
- **多 provider 配置页**: 设置页顶部新增「TTS Provider 配置」下拉框，可独立切换编辑 MiniMax 或 Qwen3 的模型和参数，不再依赖主 API 页当前选中项。模型、默认 voice_id 等核心参数按 provider 分别保存到 api.yaml。MiniMax 特有的功能开关（提示词约束、语气标签保护）和提示词模板区仅在 MiniMax 视图下显示。
- **声音复刻适配**: 上传按钮根据当前 provider 自动选择 MiniMax 语音克隆或 Qwen 声音复刻（voice enrollment）API。

## 0.9.7 (2026-05-14)

- **设置页布局**: 将模型选择和默认 voice_id 移到设置页上方，进入页面后优先配置核心语音参数。
- **独立保存**: 移除底部全局 `保存配置` 按钮，模型、默认 voice_id、角色 voice_id、本地参考音频、参考音频语言和提示词约束开关改为修改后自动保存。
- **开关交互**: 提示词约束从复选框改为状态按钮，按钮文案会在「已开启，点击关闭」和「已关闭，点击开启」之间动态变化。
- **旧模板修复**: 忽略旧版迁移残留的 `source="default"` 隐藏模板，避免角色选择粤语等语言时被旧 `auto` 约束覆盖。
- **提示词精简**: 默认约束里的语气标签列表移除中文括号标注，只保留标签本体和用途描述；已存在的默认模板槽位会随读取自动刷新。
- **语气标签保护开关**: 功能开关区新增语气标签保护按钮，可控制插件是否接管主程序括号清理，避免 `translate` 中的 `(laughs)`、`(sighs)` 等标签在合成前被删除。
- **示例格式调整**: 四语言默认提示词里的示例统一为 `dialog` 数组结构，并按中文、日语、粤语、英语分别标注 `translate` 字段含义。

## 0.9.6 (2026-05-14)

- **设置页收束**: 移除插件页底部的基础合成参数区，保留模型、兜底 voice_id、角色 voice_id、本地参考音频和提示词模板等核心配置。
- **参考音频语言**: 角色语音配置区新增参考音频语言选项，上传本地参考音频生成 voice_id 时会作为 MiniMax `language_boost` 传入。
- **提示词模板说明**: 模板区新增说明，明确只提供中/日/粤/英四套固定模板；修改角色对应语言模板后，运行时会按主程序当前语音语言自动注入对应约束。
- **旧参数清理**: 运行时不再读取旧配置残留的语言增强、语速、音量、音高、自动克隆和降噪等隐藏生成参数，避免删除 UI 后仍被旧值影响。

## 0.9.5 (2026-05-14)

- **单角色模板注入修复**: 只有一个角色参与生成时，注入内容严格使用该角色对应语言模板，不再额外补入通用默认 guard，避免手改模板在主程序里看起来又变回默认模板。

## 0.9.4 (2026-05-14)

- **角色模板读取修复**: 运行时注入提示词时，优先读取当前角色在对应语言下的手改模板；如果同语言旧版隐藏模板是手改版本，也会优先使用，不再直接回退到硬编码默认模板。
- **模板编辑体验修正**: 角色模板语言下拉框刷新时默认对齐主程序当前语音语言；同一角色内保存或刷新时保留当前正在编辑的语言，避免误改到别的语言模板。

## 0.9.3 (2026-05-14)

- **注入逻辑调整**: 提示词语言不再由配置页下拉框决定，运行时会读取主程序当前 `voice_language` 并注入对应语言模板。
- **模板编辑收束**: 每个角色只显示中文、日语、粤语、英语四个固定模板语言，不再允许新建或删除提示词版本。

## 0.9.2 (2026-05-14)

- **文案修正**: 角色语音配置页只保留当前插件实际使用的源音频要求，移除「可选示例音频小于 8 秒」说明，避免用户误解。

## 0.9.1 (2026-05-14)

- **配置页调整**: 角色语音配置区新增 MiniMax 官方参考音频要求提示，强调源音频格式、时长和大小限制。
- **界面收束**: 移除角色卡参考音频路径展示、角色语音语言下拉和自动克隆缓存路径展示；新增「打开角色卡音频位置」按钮。
- **上传逻辑**: 手动上传 voice_id 现在只使用「本地参考音频」栏位，不再自动回退到角色卡参考音频。
- **隐藏状态保护**: 配置页暂时不再读写角色语音语言映射，避免旧配置在隐藏 UI 后继续影响行为。

## 0.9.0 (2026-05-14)

- **提示词模板库**: 默认模板改为中文、日语、粤语、英语四套母版；每个角色首次打开时也会自动生成四套同语种默认提示词。
- **角色级选择**: 约束版本选择现在决定实际注入的提示词语言，不再在注入时按运行时语音语言动态改写已选版本。
- **同步规则**: 保存默认模板时只同步同语种、且仍标记为 `source="default"` 的角色版本；手动保存角色版本后会变为 `source="manual"`，避免被母版覆盖。

## 0.8.2 (2026-05-14)

- **提示词调整**: 移除 `translate` 语气标签数量上限，不再限制每条台词只能使用 1-2 个标签，改为按台词情绪自然使用。

## 0.8.1 (2026-05-14)

- **提示词优化**: 默认约束按中文、日语、粤语、英语分别生成语言适配规则，明确 `speech` 只用于屏幕显示，`translate` 才用于 Cloud TTS 合成。
- **语气标签**: 标签清单补全为 MiniMax 支持的完整标签集，并为每个标签增加中文含义和推荐使用场景。
- **生成约束**: 增加每条 `translate` 最多使用 1-2 个语气标签、标签贴近情绪位置、情绪不明显时不强行加标签等规则。

## 0.8.0 (2026-05-14)

- **功能**: 插件角色语音配置新增「角色语音语言」，可为每个角色指定中文、日语、粤语、英语或跟随主菜单。
- **提示词**: 默认 Cloud TTS 约束改为强制输出 `translate` 字段；中文语音时 `translate` 作为中文 TTS 文本，日语/粤语/英语语音时按目标语言生成。
- **修复**: Cloud TTS 启用时保护 MiniMax 支持的括号语气标签，避免主程序 `remove_parentheses()` 把 `(laughs)`、`(sighs)` 等标签从 `translate` 中删掉。
- **适配**: 角色语音语言会优先决定 MiniMax `language_boost`，并兼容主程序传入的 `zh_CN` 等中文语言码。

## 0.7.9 (2026-05-14)
