# Cloud TTS 插件更新日志

## 0.7.4 (2026-05-14)

- **修复**: 回退 0.7.2 中 `load_plugin_base_config()` 读取顺序改动 — SDK 的 `plugin_root` 是数据目录而非包目录，读包目录优先会导致永远读不到用户的 `auto_prompt_constraint`

## 0.7.3 (2026-05-14)

- **修复**: 提示词约束注入开关关闭后仍会注入 — `_sync_template_text()` 在 `wants_constraint=False` 时缺少提前返回，导致落到底部注入逻辑

## 0.7.2 (2026-05-14)

- **修复**: `load_plugin_base_config()` 读配置优先级与 `save_plugin_config()` 写目标不一致，改为包目录优先

## 0.7.1 (2026-05-14)

- **功能开关区新增 TTS 分句提示**：引导用户前往主菜单 API 设置页关闭「启用分句合成语音」(tts_split_enabled) 后点击【保存配置】

## 0.7.0 (2026-05-14)

- **移除 Paragraph 分段功能**：主程序已内置 TTS 分句开关（`tts_split_enabled`）
- 删除 `paragraph.py`、相关 state 函数、UI 开关
- 使用提示更新：引导用户在 API 设置页配置分句

## 0.6.6 (2026-05-13)

- 新增「重置默认模板」按钮，一键恢复硬编码原始默认模板
- 重置后自动全局同步所有跟随母版的角色版本

## 0.6.4 (2026-05-13)

- 修复: 新增角色卡后插件列表不实时更新 — 页签切回时自动刷新角色

## 0.6.3 (2026-05-13)

- 保存默认模板时通过 `propagate_default_template()` 全局同步
- 新建版本 `source="default"` 跟随母版；手动保存后 `source="manual"` 停止同步

## 0.6.2 (2026-05-13)

- 默认模板改为母版模式：仅 v1 一个版本，禁用新建/删除按钮
- 新增 `state.get_default_template_text()`：优先读 `默认模板.json`，回退硬编码
- 其他角色新建版本从默认模板文件读取，确保跟随母版更新

## 0.6.1 (2026-05-13)

- 「默认模板」从约束版本下拉框移到角色下拉框，作为虚拟角色
- 移除版本下拉框内 `__default__` 特殊处理逻辑

## 0.6.0 (2026-05-13)

- **角色下拉框新增「默认模板」虚拟角色**
- 新建版本写死从硬编码默认模板起步，不再读编辑框当前内容

## 0.5.2 (2026-05-13)

- **修复**: 约束版本下拉框偶尔需要双击才能选中（减少 UI 线程磁盘 I/O）
- 约束版本切换时避免重复读取 JSON 文件
- 仅在版本确实变化时才写入 selected_version，不再每次浏览都写盘

## 0.5.1 (2026-05-13)

- 约束版本新增「版本名称」字段，用户可自定义名称
- 下拉框显示改为 `v1 / 名称` 格式，不再截取内容预览
- 修复: 模板角色切换时约束版本列表不跟随刷新的问题

## 0.5.0 (2026-05-12)

- **架构变更**: 提示词约束从全局 key 字典改为**按角色独立版本管理**
- 每个角色拥有独立的约束版本文件 (`prompt_constraints/{角色名}.json`)
- 约束版本管理 UI 从 QListWidget+弹窗 重构为 QComboBox+QTextEdit 内联编辑
- 支持新建/保存/删除约束版本，至少保留一个
- 自动为每个角色创建默认 v1 模板
- 注入逻辑: 模板中所有选中角色共享同一约束文本时才注入，否则移除

## 0.4.0 (2026-05-11)

- 新增可自定义的提示词约束功能
- 支持全局约束 key 字典配置
- 新增约束新建/编辑/删除弹窗

## 0.3.0

- 合并 Paragraph 分段合成功能到本插件
- MiniMax 设置页独立于主 API 页

## 0.2.0

- 新增 voice_id 导入/导出功能
- 新增参考音频自动克隆 voice_id
- voice_id 版本管理（按角色记录历史 voice_id）

## 0.1.0

- 初始版本
- Cloud TTS 基础合成参数配置
- 模型选择、语言增强、采样率/比特率/声道
- 角色 voice_id 绑定
